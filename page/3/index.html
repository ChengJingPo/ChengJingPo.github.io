kvo 的探究
kvo :
1.添加观察者。 addObserver:forKeyPath: options: contex:
(1)即对象（就是该方法observerValueForKeyPath: ofObjectChange: context: 写在哪个类的方法里）
（3）options : NSKeyValueObservingOptionNew | NSKeyValueObservingOptionOld
（4）context:可以作为标志符（用于对对象的多个属性添加观察者时做区分）
2.监听
3.删除观察者（注：一般情况下在dealloc中释放观察者对象）

设置属性值
（1）key
（2）keyPath
KVC :
设置属性值

  • (void)setValue:(id)value forKey:(NSString*)key;
    访问属性值
  • (id)valueForKey:(NSString*)key;
    “KVC在执行KVC方法时，会优先判断是否有对应的setter,getter方法“

kvo发送通知的时机
（1）使用setter方法
（2）使用KVC的setValue
(3)手动控制发送通知（手动触发kvo前需要先关闭自动发送通知）—-模拟系统的自动发送通知，需要重写setter方法

kvo原理分析
核心：当一个类有观察者时，会生成一个中间类，并且会把该类的isa指向中间类

原理：
1.当一个object有观察者时，动态创建该object的类的子类；
2.对于每个被观察的property,重写其setter方法；
3.在重写的setter方法中，调用- willChangeValueForkey: 和 - didChangeValueForKey: ;

查看类的变化状态：
1.方法列表，属性列表，变量列表；
2.类名class，isa,object-getclass
3.setter方法的IMP，class的IMP
《重写了setter方法，class方法》

手动调用kvo

-(void)willChangeValueForKey: //属性值将要变化
-(void)didChangeValueForKey: //属性值已经改变
+(BOOL)autoMaticallyNotifiesObserversForKey: //给系统自动发送通知操作加一个开关，通过key ,判断哪些允许自动发送通知，哪些不允许

（1）手动触发KVO前，需要先关闭系统的自动发送通知；
（2）模拟系统的自动发送通知，需要重写setter方法；
（3）发送2个通知，需要嵌套：willChange 和 didChange
（4）模拟系统KVO的实现：
a.添加通知：cf_ddObserver…
b.监听通知：cf_observerValueForKey..
c.删除通知：cf_removeObserver…

重写该3个方法，关键工作：
1）注册类，集成self
2）修改isa
3)重写setter方法
4）重写class方法
5）通知外界工作

核心：
（1）修改isa;
(2)添加修改后的setter方法
（3）添加修改后的class方法
（4）通知外界
